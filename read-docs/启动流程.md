# 启动流程

按照`proc.c`中所说, 

1. call osinit
2. call schedinit
3. make & queue new G
4. call runtime·mstart

The new G calls runtime·main.

看样子是先创建任务对象g, 再启动m去执行ta.

我想这个流程实际是写在`asm_adm64.s`的第一个函数`_rt0_go`中的.

rt0_linux_amd64.s -> _rt0_amd64_linux()

rt0_linux_amd64.s -> main()

asm_amd64.s -> _rt0_go()
